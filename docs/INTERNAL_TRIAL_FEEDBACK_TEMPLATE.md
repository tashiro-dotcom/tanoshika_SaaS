# INTERNAL_TRIAL_FEEDBACK_TEMPLATE

社内トライアル用の記録テンプレート。
操作中の感想ではなく、再現可能な事実を残すことを優先する。

## 記入ルール

1. 1件につき1レコードで書く
2. 「不満」だけで終わらせず、何をしようとしていたかを書く
3. 主観は残してよいが、先に事実を書く
4. 優先度は `P1/P2/P3` で分類する

## 優先度の目安

- `P1`: 業務が止まる、誤操作しやすい、データ誤りの恐れがある
- `P2`: 作業は続けられるが、毎回迷う・手数が多い
- `P3`: 見た目、文言、軽微な違和感

## エンジニアリング原則ベースの確認観点

### 1. Learnability

- 初回で何を押せばよいか分かるか
- ラベルと実際の動作が一致しているか
- 業務用語が現場の言い方と合っているか

### 2. Efficiency

- 1つの目的に対して操作回数が多すぎないか
- 同じ情報を何度も入れさせていないか
- 一覧からそのまま次の操作に進めるか

### 3. Feedback

- 押した直後に結果が分かるか
- 成功/失敗の理由が理解できるか
- 次に何をすればよいか画面から判断できるか

### 4. Error Tolerance

- 間違えたときに復旧しやすいか
- MFA期限切れ、不正ID、未退勤などのエラーが理解できるか
- 誤操作で他人のデータを触れないか

### 5. Operational Fit

- 実際の現場順序に合っているか
- 1日1回ではなく毎日何度も使える導線か
- 紙・口頭・Excelへ戻りたくなる場面がないか

## 記入例

- 実施日: 2026-03-10
- 実施者: 事業所管理者A
- 対象環境: 社内LAN / あなたのPC
- 対象機能: 打刻
- 優先度: P2
- やろうとしたこと: 利用者の退勤打刻
- 実際に起きたこと: どの行が勤務中か一瞬で判断できなかった
- 期待していたこと: 勤務中の行だけ強く目立ってほしかった
- どこで迷ったか: 状態バッジの見分け
- 回避方法があったか: 最新打刻列を見れば判断できた
- 再現手順: 2人以上の利用者を表示して勤怠管理を開く
- 補足メモ: 勤務中だけ背景色を変えると改善しそう
